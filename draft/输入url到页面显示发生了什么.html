输入url到页面显示发生了什么

从输入url到页面显示发生了？ 是一个常见的问题，因为可全面考察 web 开发相关的知识点，这也是 web 系统的一个完整使用流程，知乎上有一篇文章当你在浏览器中输入 baidu.com 并且按下回车后发生了什么？写得很全面，本文在其基础上进一步总结，目的是加深个人的理解。

流程

解析 URL

浏览器解析输入的 url，得到协议类型，域名和资源路径。如果无法解析到一个正确的 url，就调用默认搜索引擎进行搜索。

检查 HSTS

如果 url 的协议为 http，浏览器检查自带的「预加载 HSTS（ HTTP 严格传输安全）」列表，这个列表里包含了只使用 HTTPS 进行连接的网站。如果当前 url 的域名在该列表中，就转换成 https 协议去请求。

将域名添加 HSTS 列表的方式是在第一次通过 https 访问该域名时，返回携带 Strict-Transport-Security 字段的请求。

（也可以通过重定向等方式强制转换成 https 访问）

域名解析

域名解析就是查找域名与 ip 地址的映射关系的过程，这些映射关系在 DNS 服务器其上。

  • 浏览器检查域名是否在缓存中
  • 如果没有则去本地 host 中查找
  • 如果 host 中没有则去 DNS 服务器查询

去 dns 查询通过的是 DNS 协议，本机先向本地域名服务器查询发送 DNS 请求报文，如果本地域名服务器没有记录，就向根域名服务器请求，根服务器会返回主域名服务器地址，本地域名再去主域名服务器请求得到域名服务器地址，再去请求得到 IP 地址，返回给本机。实际情况更复杂，可能会有多级域名服务器。

TCP

  • 得到IP地址后，该 HTTP 请求就会在传输层封装成 TCP
  • 然后交给网络层添加IP头部
  • 再交给数据链路层，添加 frame 头部，里面包含本地 mac 地址和网关（路由器等）的 mac 地址（mac 地址也需要通过 ARP 协议查找）

此时就构成一个封包，经过网络线路，还有各级路由器、交换机到达目的计算机，

建立一个 TCP 连接需要经过三次握手，断开 TCP 连接需要经过四次握手。

建立 TCP 链接后，开始传输数据
